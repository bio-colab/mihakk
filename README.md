# مِحَكّ · MIHAKK

### بنشمارك متانة كواشف النصوص المولّدة بالذكاء الاصطناعي أمام الهجمات العدائية
**Adversarial Robustness Benchmark for AI-Text Detectors**

</div>

---

«مِحَكّ» حجرُ الاختبار الذي كان الصاغة يمسحون عليه الذهب ليعرفوا عياره. وهذه المنصّة تفعل الشيء نفسه بالكواشف: تُخضع النصّ المولّد لهجومٍ عدائيٍّ مبنيٍّ على البحث، ثم تقيس أيُّ الكواشف صمد وأيُّها انهار — عبر أنواع النصوص والهجمات كافّة.

المنصّة **تجسيدٌ تطبيقيٌّ** لبحثٍ محكّمٍ في كشف النصوص المولّدة (تصنيف عائلات الكشف، الهجمات التي تكسرها، مصفوفة المتانة في الفصل ٨٫٨، ونتيجة الاستحالة النظرية).

---

## المبدأ الجوهري: لماذا لا تفحص المنصّة آلياً؟

أغلب الفواحص التجارية — و**Turnitin** على رأسها — **بلا واجهةٍ برمجيةٍ عامة**، وبعضها يحجب استدعاء المتصفّح (CORS). لذلك لا تستطيع أيُّ منصّةٍ أن «تفحص» النصّ آلياً بهذه الأدوات دون تحايلٍ هشّ. الصدق التقني يقتضي معماراً مختلفاً:

```
   ┌────────────┐   هجوم عدائي     ┌────────────┐   تفحصه أنت    ┌────────────┐
   │  نصٌّ مولّد  │ ───(المحرك)────► │ نصٌّ مُحوّل  │ ───(بالكاشف)──► │  نسبة بعد   │
   │ + نسبة قبل │                  │            │                │            │
   └────────────┘                  └────────────┘                └─────┬──────┘
          │                                                            │
          └──────────────► دلتا التهرّب + الاحتفاظ + الصمود ◄───────────┘
                                        │
                                        ▼
                          لوحة صدارة + مصفوفة متانة
```

**أنت الفاحص، والمنصّة أداة القياس والتحليل والترتيب.** هذا يطابق الواقع بدل ادّعاء قدرةٍ غير موجودة.

---

## المزايا

- **محرّك تعديلٍ ذكيٌّ متعدّد المزوّدين:** Claude, GPT, Gemini, Mistral, DeepSeek, GLM, MiniMax, وبوّابة OpenRouter الموحّدة. تضبط مفتاحاً واحداً أو أكثر وتبدّل بينها.
- **ستّة هجماتٍ عدائية** مبنيةٌ على البحث، منها اثنان **حتميّان يعملان بلا مفتاح**:
  - إعادة صياغة على مستوى الخطاب (DIPPER)
  - إعادة صياغة عودية (متعدّدة الجولات)
  - أنسنة (حقن تذبذب/burstiness)
  - ترجمة عكسية
  - **متجانسات بصرية (SilverSpeak)** — منفّذٌ في الكود
  - **محارف خفيّة صفرية العرض** — منفّذٌ في الكود
- **مراعاة نوع النص:** الهجوم يحقن قيود المجال (بحثي/قانوني/علمي/احترافي/تجاري/عام) في موجّه المحرك، فلا يُفسد الاقتباسات أو المصطلحات القانونية أو المعادلات.
- **لوحة صدارةٍ حيّة:** ترتيب الكواشف حسب المتانة، مقسّمٌ حسب نوع النص والهجوم، مع «أضعف أمام ماذا» و«الأمتن في كل نوع نص».
- **مصفوفة متانةٍ تفاعلية:** تجسيدٌ حيٌّ لمصفوفة الفصل ٨٫٨، تُملأ من تجاربك ومن مراجع الأدبيات.
- **مُدوّنة تجارب:** حفظٌ محليٌّ كامل، تصدير/استيراد JSON للمشاركة.
- **واجهةٌ عربيةٌ RTL** متجاوبةٌ مع كل الشاشات، بهوية «لوحة تحليل إشارة».

---

## التقنيات

- **React 19 + TypeScript + Vite**
- CSS خالص برموز تصميم (design tokens) — بلا أُطر ثقيلة، سهل التعديل.
- تخزينٌ محليٌّ عبر `localStorage` — لا خادم، لا قاعدة بيانات، خصوصيةٌ كاملة.
- استدعاء المزوّدين مباشرةً من المتصفّح.

## الخطوات داخل المنصّة

1. **المِحَكّ** ← ألصق النصّ المولّد، اختر نوعه (أو دعه تلقائياً).
2. اختر الهجوم. إن كان يحتاج محرّكاً، اضبط مفتاح النموذج في **الإعدادات**.
3. حدّد الكواشف التي استخدمتها وأدخِل نسبة الذكاء الاصطناعي **قبل** الهجوم.
4. اضغط «طبّق الهجوم» ← انسخ النصّ المُحوّل وأعد فحصه بالكاشف خارج المنصّة.
5. أدخِل النسبة **بعد** الهجوم ← احفظ التجربة.
6. راجع **لوحة الصدارة** لترى الترتيب والمصفوفة والتحليل.

## المزوّدون وتحدّي CORS

استدعاء واجهات النماذج من المتصفّح يخضع لسياسة CORS لكل مزوّد:

| المزوّد | الحالة |
|---|---|
| OpenRouter | يسمح باستدعاء المتصفّح — **الأسهل لتجربة عدّة نماذج بمفتاح واحد** |
| OpenAI · Google · Mistral · DeepSeek · GLM · MiniMax | يسمح عموماً بالاستدعاء المباشر |
| Anthropic (Claude) | يسمح عبر ترويسة `anthropic-dangerous-direct-browser-access` (مُضافة تلقائياً) |

إن واجهت حجب CORS مع مزوّدٍ ما، فأسهل حلٍّ استخدام **OpenRouter**. للحالات المتقدّمة يمكنك تشغيل بروكسي محلّي بسيط يمرّر الطلبات (خارج نطاق هذا الإصدار الأساسي).

> **ملاحظة أمنية:** المفاتيح تُحفَظ في `localStorage` بمتصفّحك فقط ولا تُرسَل إلا إلى مزوّد النموذج مباشرةً. مع ذلك، لا تستخدم هذه الأداة على جهازٍ عام، ويُفضّل مفاتيح بصلاحياتٍ محدودة.

## الهجمات الحتمية (بلا مفتاح)

هجوما **المتجانسات البصرية** و**المحارف الخفيّة** منفّذان بالكامل في `src/lib/homoglyph.ts`:

- **المتجانسات:** استبدال أحرفٍ لاتينيةٍ بأخرى تُطابقها بصرياً من جداول Unicode مختلفة (سيريلي/يوناني)، فيكسر التقطيع (Tokenization) لدى الكاشف ويرفع الحَيْرة صناعياً.
- **المحارف الخفيّة:** حقن محارف صفرية العرض (ZWSP/ZWNJ/ZWJ) بين الرموز — غير مرئيةٍ للقارئ لكنها تحطّم التقطيع.

هذان يوضّحان حرفياً هجوم SilverSpeak من الفصل ٨٫٣، ويعملان فوراً دون أي إعداد.

## التحويل إلى تطبيق Windows / Android مستقبلاً

المنصّة تطبيق ويبٍ نقيٌّ بلا اعتمادٍ على خادم، فتحويلها مباشر:

### سطح المكتب (Windows/macOS/Linux) عبر **Tauri** (موصى به — حجمٌ أصغر)

```bash
npm install -D @tauri-apps/cli
npx tauri init      # اضبط build command: "npm run build"، وdist: "dist"
npx tauri build     # يُنتج مثبّت msi/exe لويندوز
```

بديلٌ عبر **Electron**:

```bash
npm install -D electron electron-builder
# أنشئ main.js يحمّل dist/index.html، ثم:
npx electron-builder --win
```

### أندرويد عبر **Capacitor**

```bash
npm install @capacitor/core @capacitor/android
npx cap init mihakk com.biocolab.mihakk --web-dir=dist
npm run build
npx cap add android
npx cap sync
npx cap open android   # يفتح Android Studio لبناء APK/AAB
```

> **تنبيه CORS على المنصّات الأصلية:** في تطبيقات Tauri/Capacitor يمكن تجاوز قيود CORS للمتصفّح عبر طبقة HTTP الأصلية (Tauri `http` plugin أو Capacitor `CapacitorHttp`)، ما يجعل استدعاء كل المزوّدين مباشراً دون بروكسي — ميزةٌ إضافيةٌ للنسخ الأصلية.

## الخصوصية والبيانات

- كل شيءٍ محليٌّ في متصفّحك: التجارب والمفاتيح في `localStorage`.
- لا تتبّع، لا تحليلات، لا خوادم وسيطة.
- التصدير/الاستيراد عبر ملفات JSON بيدك وحدك.

---

## حدود الأداة (شفافية)

- المنصّة **لا تفحص** النصوص نيابةً عنك؛ أنت تُدخل نتائج الكواشف.
- جودة هجمات المحرّك تتبع جودة النموذج ومفتاحك.
- مراجع الأدبيات تقديريةٌ لأغراض التأسيس، وتُميَّز وتُفصَل عند إيقافها من الإعدادات.
- التصنيف التلقائي لنوع النص استدلاليٌّ بكلماتٍ مفتاحية، لا نموذج تعلّمٍ عميق.

## الأساس البحثي
